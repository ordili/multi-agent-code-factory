# Python code standards (Profile python) — language-specific; injected after dev-principles-snippet
# Universal rules: docs/design/pipeline/artifact-templates/dev-principles-spec.md
# Full Python spec: docs/design/pipeline/python-style.md

- Python >= 3.11; follow PEP 8 (enforced by Ruff).
- Format: ruff format (Black-compatible, line length 88, double quotes).
- Naming: snake_case functions/vars, PascalCase classes, UPPER constants.
- Public functions/classes: type hints + Google-style docstrings.
- Error handling: catch specific exceptions; use `raise ... from e`; never bare `except:` or silent `pass`.
- Resources: use context managers (`with`) for files, locks, and connections.
- Dependencies: prefer stdlib; add third-party packages only when spec requires them.
- Tests: pytest under tests/test_*.py; no network in unit tests unless specified.
- Logging: use logging module; no print except CLI user output.
- Before finishing: code must pass `python -m ruff check . && python -m ruff format --check .` and `python -m pytest -q --junitxml=reports/junit.xml`.
