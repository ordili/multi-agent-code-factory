时序图 — 失败 / 升环 / 触顶

用 alt 块概括异常路径；计数器与路由细节仍以「路由决策表」为准。 

sequenceDiagram
    participant G as validate / qa
    participant Ag as pm · architect · developer
    participant R as reviewer
    participant F as fail

    alt spec 或 design 校验失败
        G->>Ag: violations
        Ag->>G: 修订后再入 validate
    else QA 测试失败
        G->>Ag: test failed
        Ag->>G: 修订后再入 qa
    else Reviewer 升环
        R->>Ag: next_stage = pm / architect / developer
        Ag->>G: 从对应再入点继续
    else 循环触顶
        G->>F: fail 或 escalation_hitl
    end

路由决策表（与 graph_routing.py 一致）

route_after_requirements_validate

条件下一节点状态变更
requirements_validation.passed=false 且 block_on=error，未触顶 pm requirements_revision_count++；清空 design / dev / test / review
校验通过 + require_hitl requirements_hitl → 固定到 architect MVP 自动批准，不中断
校验通过 + 无 hitl architect —
requirements_revision_count ≥ max_requirements_revisions fail 或 escalation_hitl 由 on_limit_exceeded 决定

route_after_design_validate

条件下一节点状态变更
design_validation.passed=false 且 block_on=error，未触顶 architect design_revision_count++；清空 dev / test / review
校验通过 + hitl 需要 design_hitl → 固定到 developer MVP 自动批准
校验通过 + 无 hitl developer —
触顶 fail / escalation_hitl —

route_after_qa

条件下一节点状态变更
test_report.passed=true reviewer —
测试失败，未触顶 developer impl_retry_count++
impl_retry_count ≥ max_impl_retries fail / escalation_hitl —

route_after_reviewer

review.next_stage条件下一节点
deploy approved=true deploy_hitl → deploy → END
deploy approved=false，未触顶 developer（impl_retry_count++）
developer 未触顶 / 触顶 developer 或 fail / escalation
architect 未触顶 / 触顶 architect（design_revision_count++）或 fail
pm 未触顶 / 触顶 pm（requirements_revision_count++）或 fail

默认循环上限（LoopLimits）

配置项默认值计数器字段
max_requirements_revisions1requirements_revision_count
max_design_revisions2design_revision_count
max_impl_retries3impl_retry_count
on_limit_exceededfail可选 escalation_hitl

节点 ID类型说明

requirements_validate校验规则校验 spec，写 requirements_validation
